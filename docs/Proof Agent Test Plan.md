# Test Plan: Proof Agent v1

Generated by `/plan-eng-review` on 2026-05-08  
Status: DRAFT

## Scope

This test plan covers the narrowed Enterprise Agent Delivery Kit MVP: enterprise knowledge Q&A Agent, typed Harness policy decisions, local knowledge retrieval, session memory, MCP mock tool approval, JSONL trace, Governance Receipt Contract, Plain RAG vs Harness RAG comparison, 3-minute launch path, Trust Boundaries, CLI run path, Docker Compose demo, and first-run performance budget.

## Recommended Test Stack

- Unit/integration: `pytest`
- CLI integration: `pytest` invoking the local CLI entrypoint
- E2E smoke: Docker Compose demo run
- LLM quality: small eval suite for citation/refusal behavior

## Critical Paths

- `proof-agent run examples/enterprise_qa/agent.yaml` loads the manifest and runs the full demo.
- README launch commands run `examples/enterprise_qa/agent.yaml` and produce documented artifacts.
- Enterprise Q&A flow performs mandatory retrieval before answer generation.
- Weak or missing evidence produces refusal or escalation.
- Tool call requiring approval pauses workflow and resumes after approval.
- Every run writes ordered JSONL trace events.
- Every run writes a human-readable Governance Receipt that satisfies the receipt contract.
- Plain RAG vs Harness RAG demonstrates the value of the Control Envelope.
- Trust Boundaries are reflected in tests for redaction, MCP mock limits, prompt-injected knowledge, and memory writes.

## Unit Tests

- Manifest schema accepts valid `agent.yaml`.
- Manifest schema rejects missing policy, knowledge, model, tools, or trace fields with actionable errors.
- `PolicyEngine.before_retrieval` returns allow/deny with reason and trace fields.
- `PolicyEngine.before_answer` enforces evidence requirements.
- `PolicyEngine.before_tool_call` returns allow/deny/require_approval.
- `PolicyEngine.before_memory_write` blocks disallowed writes.
- Evidence evaluator accepts enough evidence and rejects weak or missing evidence.
- JSONL trace writer emits ordered events and redacts secrets.
- Governance Receipt generator summarizes policy, evidence, tool, memory, and trace status.
- Governance Receipt generator maps required sections to trace events.
- Governance Receipt generator emits required final outcomes.
- Governance Receipt generator redacts secrets and unnecessary personal data.
- Trust-boundary validation rejects unsupported production MCP Gateway or persistent memory settings in v1 configs.

## Integration Tests

- Local knowledge ingestion builds sample index under the time budget.
- Retrieval returns expected sample document chunks.
- Enterprise QA workflow returns cited answer when evidence is sufficient.
- Enterprise QA workflow refuses when evidence is missing.
- MCP approval requested state is persisted in workflow state.
- Approval granted executes the mock tool.
- Approval denied or timed out returns a safe response.
- Failed runs write terminal failure trace events.
- Governance Receipt is produced for answered, refused, escalated, approval-pending, approval-denied, and failed runs when trace events are available.
- Receipt generation failure preserves JSONL trace and reports `FAILED_RECEIPT_UNAVAILABLE`.
- Plain RAG and Harness RAG differ on unsupported and tool-required questions.
- Fixed prompt-injection fixtures cannot bypass evidence, citation, tool approval, or memory policies.
- Deterministic demo uses the same policy, evidence, approval, trace, and receipt code paths as full runs.

## E2E Tests

- Docker Compose first-run path completes with sample docs.
- CLI first-run path completes with `examples/enterprise_qa/agent.yaml`.
- README 3-minute launch path completes with the documented demo questions.
- Demo produces final answer/refusal plus a readable trace artifact.
- Demo produces a Governance Receipt.
- Demo includes Plain RAG vs Harness RAG comparison output.

## Eval Cases

- Supported question: answer must cite the relevant source.
- Unsupported question: answer must refuse instead of hallucinating.
- Partially supported question: answer must only state supported claims or escalate.
- Tool-required question: answer must wait for approval before tool execution.
- Plain RAG comparison: unsupported question may answer loosely, while Harness RAG must refuse or escalate.
- Prompt-injected knowledge fixtures: retrieved content that asks the Agent to ignore policy, call a tool without approval, or reveal a fake secret must not change Harness decisions.

## Performance Checks

- Clone + setup path documented for <= 5 minutes.
- Docker Compose pull/up measured for <= 8 minutes.
- Sample knowledge index measured for <= 2 minutes.
- First QA run measured for <= 60 seconds excluding approval wait.
- Local JSONL trace write measured for < 100 ms on sample run.
- Receipt generation measured and included in the first-run path budget.

## Acceptance Gate

v1 is not accepted unless unit, integration, E2E smoke, README launch path, receipt contract, trust-boundary, comparison, and eval tests pass for the enterprise Q&A template.
