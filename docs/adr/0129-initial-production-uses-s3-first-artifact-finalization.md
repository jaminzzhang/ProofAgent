# Initial Production Uses S3-First Artifact Finalization

Accepted.

[FRAME | HIGH] Initial production finalizes immutable artifacts with a deliberately simple S3-first commit rather than a recoverable per-object Saga. A producer freezes the required artifact bytes, assigns unique non-overwritable keys, uploads and verifies every exact object version by length and SHA-256, and uploads a verified Production Artifact Manifest last. One PostgreSQL transaction then binds that manifest and its immutable references to the owner and records the Run terminal state or makes the Knowledge, validation, evaluation, or export result available. This PostgreSQL transaction is the only visibility point, and no success response may precede it.

[FRAME | HIGH] A process failure before the PostgreSQL visibility transaction may discard the uncommitted Attempt output and some fine-grained progress. The abandoned S3 objects remain invisible because they have no authoritative PostgreSQL reference; a reference-aware orphan collector removes them only after a bounded grace period. Initial production does not add `PENDING_UPLOAD`, `UPLOADED_UNVERIFIED`, or resumable Artifact Set commit states. A live producer may retry the final PostgreSQL transaction within its deadline, but a producer lost before commit yields an explicit failed or replacement Attempt rather than reconstructing success from unreferenced objects.

[FRAME | HIGH] PostgreSQL still durably retains the admitted Run request, idempotency identity, leases, cancellation, coarse `QUEUED`, `RUNNING`, `FINALIZING`, and terminal state, and the final artifact references. Fine-grained Run Progress Stream details are best-effort: a reconnect receives the current durable coarse state and subsequent trace-safe details, but is not guaranteed every missed intermediate event. Browser disconnection never cancels execution, raw model output remains hidden until validation, and a committed governed terminal result must always resolve to its verified final artifacts. An Attempt lost before that visibility commit may terminate with an explicit PostgreSQL infrastructure-failure record and no audience result; it cannot be represented as a committed governed result. This paragraph partially supersedes ADR-0119's requirement to persist and replay every ordered progress event.
