# Validation Capture Owns Stage Failure Diagnostics

Accepted.

Validation Capture Dashboard Reveal will show stage failure diagnostics from the Sensitive Validation Capture Artifact instead of reconstructing them from ordinary trace events in Dashboard. The artifact may include a validation-safe **Workflow Stage Failure Diagnostic Projection** with stable error codes, role names, bounded lengths, stage status, trace event references, and bounded contract-field diagnostics, while continuing to exclude provider response bodies, raw failed outputs, rejected field values, raw Prompt or context payloads, Runtime Plane state, free-form exception messages, stack traces, secrets, and chain-of-thought. This is a compatible `validation_capture.v2` extension with a default-empty `failure_diagnostics` section rather than a `validation_capture.v3` migration. Existing v2 artifacts without this section remain readable but are not backfilled or synthesized from trace; Dashboard should mark diagnostics as not recorded and invite a rerun with full capture when needed. Failure diagnostics remain independent execution facts rather than fields inside Workflow Stage Result Summary, so stage summaries continue to describe governed stage output while diagnostics explain repairable exceptional stops; governed refusals, clarification needs, and approval pauses remain in their existing projections. The diagnostic payload stores stable fields only, and Dashboard maps those fields to fixed safe explanatory copy; contract failures may include contract name, stable violation codes, bounded field paths, and violation counts, but not raw validation error text or rejected values. This keeps validation review under the same `agent.validate` permission, retention, and payload safety gate as the rest of `validation_capture.v2`, and prevents frontend trace parsing from becoming a second governed-semantics interpreter.
