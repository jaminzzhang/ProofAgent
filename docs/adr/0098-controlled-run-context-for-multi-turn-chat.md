# Controlled Run Context For Multi-Turn Chat

Proof Agent will support multi-turn Chat through a Control Plane-owned **Context Assembler** that builds **Controlled Run Context** for each governed run, instead of relying on raw transcript injection, a single summary field, runtime adapter stitching, or model-provider memory behavior. The model separates **Conversation Timeline**, **Working Context**, **Conversation Compaction**, and **Controlled Agent Memory**: the full timeline remains the record, Working Context is the bounded model-facing subset, compaction summarizes older turns with provenance when context budget is exceeded, and memory stores only governed, structured, scoped, deletable facts.

This preserves the core Harness invariant that every turn remains its own governed run with retrieval, evidence admission, policy checks, validation, trace, and receipt. Conversation recall may answer questions about prior turns, but business fact follow-ups still require Knowledge or Tool evidence. Clarification continuation is a new governed run only after a prior relevant run ended with `WAITING_FOR_USER_CLARIFICATION`; ordinary user supplements after answered or refused runs must go through Intent Resolution rather than inheriting the previous task goal. Working Context uses cache-stable section ordering, placing stable Harness, Agent, policy, tool, and stage context before frequently changing evidence, memory, recent turns, and compaction summaries.

Ordinary observability records only a **Trace-Safe Context Assembly Summary** with source references, included turn ids, memory ids, compaction summary ids, budget decisions, and fallback status. Complete Working Context or detailed context assembly content may be captured only through **Sensitive Context Capture** with explicit authorization, safety gates, and short retention.

Considered alternatives were raw chat transcript injection, provider-native memory, a single `ContextAdmission.summary`, and treating memory as business evidence. We reject them because they blur evidence authority, make audit and deletion boundaries weak, reduce prompt-cache stability, and let runtime or provider mechanics redefine Harness governance semantics.
