# Hybrid Retrieval Separates Relevance Authority And Admission

Accepted.

[FRAME | HIGH] Control Plane builds a Governed Hybrid Retrieval Request from one Resolved Hybrid Knowledge Binding, its pinned Knowledge Retrieval Profile Revision, Institution Authorization Context, approved applicability metadata, validated user conditions, and bounded query budgets. Before content retrieval, Hybrid Index Provider verifies the Knowledge Index Projection Attestation against exact index identity, generation, Source Publication Sequence, and S3 Rule Unit Publication Manifest digest. It then applies approved visibility and publication filters before OpenSearch BM25 and dense-vector retrieval, uses Source-local reciprocal-rank fusion, sends only authorized and manifest-matching candidates to the private reranker, and performs bounded Rule Unit Context Expansion in which every added rule, definition, heading, or table context independently passes the same filters. OpenSearch and reranker values remain Provider-Native Relevance Scores only; Insurance Rule Authority Gate independently verifies Rule Unit Revision and authoritative manifest digests, version, applicability, precedence, authorization, and citation, while Evidence Admission independently determines model context eligibility. Insurance Rule Comparison Queries also require every Comparison Required Evidence Slot to be satisfied. We accept separate stages and scores to prevent retrieval relevance from masquerading as business authority or answer confidence.
