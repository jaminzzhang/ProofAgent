# Untrusted Web Search Tool

Accepted.

Proof Agent will provide realtime public web search as an **Untrusted Web Search Tool** behind ToolGateway, not as a Knowledge Source, retrieval provider, Skill, MCP-first integration, or browser automation path. Search results become **Untrusted Web Context** and may be projected into a fixed **Untrusted Web Supplement** section, but they do not become Candidate Evidence, cannot pass Evidence Admission, cannot satisfy citation gates, and cannot change the governed outcome from `REFUSED_NO_EVIDENCE` to `ANSWERED_WITH_CITATIONS`.

V1 uses a provider-neutral local Tool Handler adapter so deployments can connect different search vendors later without changing Agent Contract semantics, trace schema, response projection, or evaluation taxonomy. The tool returns bounded search-result snippets through dedicated untrusted-web contracts instead of raw dictionaries or `EvidenceChunk` objects, and it does not fetch or render full web pages.

Before the tool receives a query, the Harness applies deterministic **Web Search Query Sanitization** that replaces sensitive values with safe placeholders; LLM-based query rewriting is excluded from V1. Chat surfaces may expose a per-run or per-conversation "network supplement" toggle, defaulting off, but execution still requires the Agent to explicitly declare the tool, policy to allow it, sanitization to pass, and the run budget to permit it. Trace records structured summaries such as sanitization categories, sanitized query, provider, result count, domains, URLs, latency, and errors, while Governance Receipt records only a compressed disclosure that web search was used and was not counted as controlled evidence.

Search vendor configuration is a Dashboard-managed Tool Source using a trusted, plugin-like Tool Source Descriptor. V1 descriptors are registered in framework code; Dashboard users can create Tool Source instances from descriptor-declared configuration fields and environment-variable credential references, but cannot upload executable adapter code. Agent Tool Bindings constrain how an Agent may use the Tool Source, including allowed tool contracts, call budgets, parameter bounds, fallback usage, approval behavior, and customer-facing disclosure.

Unlike Knowledge Source publications, Tool Source configuration is a live shared connection and is not pinned into Published Agent Versions. We accept this lower-complexity trade-off because search vendor configuration is operational infrastructure rather than controlled evidence. Runtime trace and Governance Receipt must record the resolved Tool Source id, provider, configuration revision or update timestamp, credential env ref name, endpoint host, timeout, budget, and a disclosure that exact historical replay is not guaranteed.
