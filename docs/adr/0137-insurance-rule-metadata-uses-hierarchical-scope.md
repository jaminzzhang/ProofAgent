# Insurance Rule Metadata Uses Hierarchical Scope

Accepted.

[FRAME | HIGH] Insurance Knowledge uses document-level Approved Insurance Rule Metadata as the default scope and creates finer Insurance Rule Units only where a reviewed section, clause, table row, or row group changes applicability, precedence, or meaning. Fine-grained units inherit document scope and store only business-reviewed overrides; isolated table cells and fixed-size retrieval chunks never become rule authority by themselves, although cell coordinates remain citation and table-context anchors. Each published Insurance Rule Unit Revision immutably combines canonical content and lineage, Approved Insurance Rule Metadata revision, and Approved Insurance Knowledge Visibility Scope revision, so any approved content, authority, or visibility change creates a new revision rather than rewriting historical search facts. This supports a corpus that is mostly uniform at document level but contains material section- and table-level exceptions. We accept a hierarchical metadata model and selective review complexity to avoid both duplicating metadata across every chunk and flattening exceptional rules into an incorrect whole-document scope.
