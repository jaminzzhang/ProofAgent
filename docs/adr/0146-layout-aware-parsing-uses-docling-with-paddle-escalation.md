# Layout-Aware Parsing Uses Docling With Paddle Escalation

Accepted.

[FRAME | HIGH] Expanded insurance Knowledge uses Docling Standard Pipeline without a VLM path as its primary layout-aware parser and escalates OCR-required or structurally suspect pages to a pinned PaddleOCR PP-StructureV3 service. Both adapters map into one immutable provider-neutral Structured Knowledge Document Artifact containing page, block, heading, table-cell, coordinate, reading-order, and lineage facts; vendor JSON and Markdown remain diagnostic projections rather than authority. Deterministic quality rules or human review select one retained result at a recorded block or table boundary; a block never silently mixes both parsers, and uncertain cross-page table reconciliation enters review. Structured Knowledge Artifact Build Identity combines source-byte hash with exact library, model, weight, OCR, preprocessing, configuration, and canonical-schema fingerprints without changing the source-byte Knowledge Document Revision identity. We accept two parser adapters because the corpus is dominated by layout-complex native PDFs but contains a material OCR and complex-table tail that one unverified parser path cannot safely cover.
