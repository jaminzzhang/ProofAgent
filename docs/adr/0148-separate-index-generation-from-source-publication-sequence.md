# Separate Index Generation From Source Publication Sequence

Accepted.

[FRAME | HIGH] Hybrid Index Provider separates rebuild-heavy Knowledge Index Generation from routine Knowledge Source Publication Sequence. Canonical artifact schema, Rule Unit search projection, OpenSearch mapping or analyzer, embedding revision, instruction, dimension, pooling, normalization, or another stored index-compatibility change creates a new generation and requires the applicable evaluation gate. A parser pipeline change always creates a new Structured Knowledge Artifact Build Identity and requires parser plus affected retrieval evaluation, but creates a new Index Generation only when it changes a stored compatibility field. Ordinary document, Approved Insurance Rule Metadata, or Approved Insurance Knowledge Visibility Scope changes reuse a compatible generation, create new immutable Insurance Rule Unit Revisions where authority facts changed, and update only affected Rule Unit Publication Membership intervals while preserving earlier sequence results. One durable Source-scoped publication attempt owns a fencing token and attempt-scoped idempotent OpenSearch writes; final PostgreSQL commit takes a short Source-row lock and verifies the live attempt, one-use validation, Draft and candidate digests, generation, exact S3 Rule Unit Publication Manifest, and a new immutable Knowledge Index Projection Attestation chain entry before exposing the sequence. That entry descends from the prior same-generation proof and covers every retained sequence, allowing historical bindings to accept a verified descendant without changing corpus identity. Published hybrid Agent resolution pins source snapshot, index generation, publication sequence, publication-time attestation, and the separately versioned Knowledge Retrieval Profile Revision. A failed attempt leaves only unreferenced derived projection state and may consume a harmless sequence gap, while prior sequences remain queryable. We accept interval bookkeeping and serialized publication to meet incremental activation and historical replay without duplicating a million-page index for every publication or querying mutable latest state.
