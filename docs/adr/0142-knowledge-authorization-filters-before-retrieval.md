# Knowledge Authorization Filters Before Retrieval

Accepted.

[FRAME | HIGH] One Published Knowledge Source may contain insurance rules with different institution, region, channel, role, or business-line visibility. Approved Insurance Knowledge Visibility Scope declares explicit `PUBLIC` or `RESTRICTED` visibility and explicit `ALL` or `ALLOWLIST` mode for every restricted dimension; missing or unapproved scope blocks publication, allowlist values are ORed within a dimension, dimensions are ANDed, and runtime intersects the approved scope with Institution Authorization Context. Each governed run derives an Authorized Insurance Knowledge Scope and excludes unauthorized Knowledge Documents and Insurance Rule Unit Revisions before lexical retrieval, vector retrieval, reranking, Rule Unit Context Expansion, or model-visible context construction. Every neighboring rule, definition, heading, and table-context unit added during expansion must independently pass the same publication and visibility filter before content is read. Post-retrieval filtering and prompt instructions are not authorization controls, and trace-safe routing summaries must not disclose identities or metadata of excluded rules. We accept stricter metadata and search-filter requirements because retrieving sensitive content before deletion would already cross the authorization boundary.
