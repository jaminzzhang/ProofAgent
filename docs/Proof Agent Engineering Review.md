## Step 0: Scope Challenge

The original framework-first design was too broad for a first public release. The accepted scope is now **narrowed Enterprise Agent Delivery Kit MVP**: build the minimum framework surface needed to deliver one strongly controlled enterprise knowledge Q&A Agent.

The v1 scope includes typed policy decisions, LangGraph-backed workflow execution, local knowledge, session memory, one MCP mock tool, explicit approval state, JSONL trace, Governance Receipt Contract, one top-level `agent.yaml`, Plain RAG vs Harness RAG comparison, 3-minute launch script, Trust Boundaries, CLI execution, Docker Compose, and CI.

## What Already Exists

- LangGraph already covers workflow execution, persistence/checkpointing, interrupts, and human-in-the-loop patterns. Reuse it as the only public v1 runtime.
- LlamaIndex or equivalent libraries already cover ingestion, indexing, retrieval, and vector store integration. Reuse local ingestion and vector search first.
- MCP already standardizes tool/resource access. Use a mock MCP tool to prove controlled invocation rather than designing a full gateway first.
- Langfuse/OpenTelemetry can receive traces later. Do not make either the v1 audit source of truth.

## NOT In Scope For v1

- Public multi-runtime support. Keep runtime adapter boundaries internal.
- Multiple production-grade knowledge providers. Start with local docs and local vector search.
- User/task memory stores beyond session memory.
- GUI policy playground or visual approval console.
- Full Langfuse/OpenTelemetry integration as a required dependency.
- Full production MCP Gateway or OAuth authorization implementation.
- 3-5 templates. Ship enterprise knowledge Q&A first; additional templates are deferred until the delivery kit is proven.

## Architecture Decisions

1. **PolicyEngine contract first:** typed decisions `allow`, `deny`, `require_approval`, and `escalate`, each with reason and trace metadata.
2. **Policy enforcement points:** `before_retrieval`, `before_answer`, `before_tool_call`, and `before_memory_write`.
3. **Runtime boundary:** LangGraph is public v1 runtime; runtime adapters stay internal until demand exists.
4. **Tool approval:** approval is a workflow state, not a hidden callback.
5. **Evidence contract:** weak or missing evidence causes refusal or escalation.
6. **Audit source of truth:** JSONL trace first, adapters later.
7. **Template config:** one top-level `agent.yaml` references policy, knowledge, tools, and trace settings.
8. **Governance Receipt Contract:** every run produces a human-readable summary derived from trace events, with required sections, outcome enum, redaction behavior, and tests.
9. **Plain RAG comparison:** v1 includes a comparison path that shows the difference between loose RAG and Harness-controlled RAG.
10. **Launch script:** the README path is a smoke-tested demo contract, not marketing copy.
11. **Trust Boundaries:** v1 documents security boundaries before implementation starts.
12. **Performance budgets:** first-run and runtime budgets are engineering requirements.

## Failure Modes

| Failure mode | Test required | Error handling | User-visible behavior |
| --- | --- | --- | --- |
| Manifest references missing policy file | Unit/integration | Fail fast | Actionable config error |
| Knowledge index missing or empty | Integration | Refuse answer | Clear "no evidence" message |
| Retrieval timeout | Unit/integration | Refuse or retry once | Clear timeout/refusal message |
| Weak evidence returned | Unit/eval | Evidence rejection | Refusal or escalation |
| Tool approval denied | Integration | Continue safely | Explain tool was not run |
| Tool approval timeout | Integration | Safe terminal state | Timeout message + trace event |
| Memory write violates policy | Unit | Deny write | Continue without storing |
| Trace write fails | Unit/integration | Fail closed or emit local fallback | Clear run failure |
| Governance Receipt generation fails | Unit/integration | Preserve JSONL trace and emit `FAILED_RECEIPT_UNAVAILABLE` | Clear audit artifact error with trace path |
| Sensitive data appears in receipt candidate | Unit/integration | Redact or fail closed | Receipt names redacted field class only |
| Prompt-injected knowledge asks to bypass policy | Eval/integration | Ignore instruction and enforce policy | Refusal, escalation, or cited answer only |
| LLM returns uncited claim | Eval/integration | Refuse or repair | No unsupported final answer |

Critical gaps: all failure modes are currently unimplemented and untested because the repo is still documentation-only.
